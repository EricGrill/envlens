# EnvLens Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build EnvLens v0.1.0 — a Rust TUI/CLI that inspects, compares, and debugs environment-variable configuration across dotenv files, Docker Compose, package scripts, CI configs, and the process environment — per the approved spec.

**Architecture:** Single crate, strict layering: a pure deterministic `core` pipeline (`scan → parse → resolve → diagnose`) consumed by three thin frontends (TUI, `check`, `report`). External facts (git-tracked files, process env) are captured by frontends and passed into `core::analyze` as data.

**Tech Stack:** Rust (edition 2024, stable toolchain), ratatui + crossterm, clap 4 (derive), serde/serde_yaml/serde_json, ignore, regex, anyhow/thiserror. Dev: insta, assert_cmd, predicates, tempfile.

**Spec:** `docs/superpowers/specs/2026-07-05-envlens-design.md` (all §N references below point there). Read it before starting any chunk.

**One deliberate spec refinement (applies to every chunk):** the spec says core's only env I/O is `std::env::vars_os()`. Rust edition 2024 makes `std::env::set_var` unsafe, so unit tests could not safely control a process-reading core. We therefore inject the process snapshot the same way as `tracked_files`: frontends call `core::parsers::process::capture()` and pass the result in. Core ends up with file reads only — strictly purer than the spec's claim, no behavior change. The `analyze` signature everywhere in this plan is:

```rust
pub struct External {
    pub process_env: BTreeMap<String, String>,        // from parsers::process::capture()
    pub tracked_files: Option<BTreeSet<PathBuf>>,     // from git::tracked_files(); None disables SecretInTrackedFile
}
pub fn analyze(root: &Path, config: &Config, profile: Option<&str>, external: External) -> Result<Analysis, AnalyzeError>
```

`AnalyzeError` has exactly one variant family: the root is missing/unreadable (→ exit 3, §11). Everything else degrades to per-source `ParseError`s + diagnostics (§13).

**Conventions for every task:** TDD (write the failing test first, watch it fail, implement, watch it pass); commit after every green step with a conventional-commit message; `cargo fmt` + `cargo clippy --all-targets -- -D warnings` must be clean before each commit. Run `cargo test` (whole suite) before each commit, not just the new test.

```rust
pub type SourceId = String;

#[derive(..., Copy, PartialOrd, Ord)]
pub enum SourceKind { Dotenv, DotenvExample, Compose, PackageScript, Process, Ci }

pub struct ParseError { pub line: Option<u32>, pub message: String }

pub struct EnvSource {
    pub id: SourceId,                 // ".env", "apps/web/.env", "docker-compose.yml[api]", "package.json[dev]", "process"
    pub kind: SourceKind,
    pub path: Option<PathBuf>,        // project-relative; None for process
    pub context: Option<String>,      // service or script name
    pub precedence: u32,              // final resolved rank; higher wins (see Task 12)
    pub enabled: bool,
    pub errors: Vec<ParseError>,
}

#[derive(..., Copy)]
pub enum SecretClass { None, KeyLike, ValueLike, Both }
impl SecretClass { pub fn is_secret(self) -> bool { !matches!(self, Self::None) } }

pub struct VariableOccurrence {
    pub key: String,
    pub raw_value: Option<String>,    // None only for bare compose list keys
    pub parsed_value: Option<String>,
    pub source_id: SourceId,
    pub line: Option<u32>,
    pub is_empty: bool,
    pub is_inherited: bool,
    pub no_expand: bool,              // true for single-quoted dotenv values (§7: no expansion)
    pub secret: SecretClass,
}

#[derive(..., Copy, PartialOrd, Ord)]
pub enum Severity { Info, Warning, Error }

#[derive(..., Copy)]
pub enum DiagnosticCode { DuplicateKey, ConflictingValues, MissingRequired, EmptyRequired,
    UndefinedReference, CircularReference, InvalidDotenvLine, SecretInTrackedFile,
    InheritedUnresolved, ShadowedValue }

pub struct Diagnostic { pub severity: Severity, pub code: DiagnosticCode, pub message: String,
    pub key: Option<String>, pub source_id: Option<SourceId>, pub line: Option<u32> }

pub struct VariableSummary {
    pub key: String,
    pub effective: Option<(String, SourceId)>,
    pub occurrences: Vec<VariableOccurrence>,   // precedence ascending
    pub diagnostics: Vec<Diagnostic>,
    pub is_required: bool,
    pub is_missing: bool,
    pub is_secret_like: bool,
}

pub struct Analysis {
    pub root: PathBuf,
    pub profile: String,              // "default" when none selected
    pub sources: Vec<EnvSource>,      // precedence ascending; disabled sources retained
    pub variables: Vec<VariableSummary>, // sorted by key
    pub diagnostics: Vec<Diagnostic>, // all, ordered: severity desc, then key, then source_id
}
```
